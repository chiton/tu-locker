## Why

tu-locker necesita una presencia web profesional para comunicar sus servicios de guardamuebles y self-storage en Bahia Blanca, generar confianza y facilitar consultas comerciales por WhatsApp, telefono y formulario.

El sitio debe estar listo para deploy, con contenido en espanol, navegacion clara, diseno responsive y estructura tecnica mantenible para futuras mejoras.

## What Changes

- Crear un sitio institucional completo para la empresa `tu-locker`, con dominio objetivo `tu-locker.com.ar`.
- Presentar servicios de alquiler de espacios de almacenamiento para objetos personales, muebles, mudanzas y uso comercial.
- Incluir secciones diferenciadas de Inicio, Sobre Nosotros, Ubicacion, Seguridad, Tamanos de Storage, Planes y Formas de Pago, Como acceder y Contacto.
- Agregar un banner principal con mensaje claro, por ejemplo "Espacio seguro para lo que mas valoras", y llamadas a la accion como "Ver planes" y "Solicitar presupuesto".
- Agregar mapa embebido para Montevideo 1450, Bahia Blanca, Buenos Aires, Argentina, junto con informacion de acceso.
- Agregar llamadas a la accion hacia WhatsApp `+54 291 4239860`, telefono, formulario y email `contacto@tu-locker.com.ar`.
- Incluir testimonios, preguntas frecuentes, animaciones suaves y boton flotante de WhatsApp cuando no agreguen complejidad innecesaria.
- Optimizar metadata basica SEO con titulo y descripcion para `tu-locker.com.ar`.
- Inicializar o completar una aplicacion Next.js App Router con TypeScript y Tailwind CSS, siguiendo las practicas de `AGENTS.md` para estructura, estilo y buenas practicas.

## Impact

- Crea la aplicacion web principal del repositorio si aun no existe.
- Afecta archivos de configuracion de Next.js, TypeScript, Tailwind y scripts de npm.
- Agrega componentes reutilizables en `components/` y contenido de pagina en `app/`.
- No requiere backend persistente ni integraciones pagas para la primera version.
- El formulario de contacto puede implementarse inicialmente con comportamiento estatico o enlace de contacto, sin almacenamiento de datos.
- Debe mantener codigo limpio, organizado, escalable y comentado solo cuando aporte claridad.
